## User Stories

User stories describe features from the user's perspective. They follow the format:
> "As a [user], I want [capability] so that [benefit]"

## PRDs (Product Requirements Documents)

PRDs provide comprehensive feature specifications for major capabilities.

*PRDs will be added as major features are specified.*

---

## Adding Requirements

### User Story Format

```markdown
# US-X.X: [Title]

## User Story
As a [user type], I want [capability] so that [benefit].

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2

## Notes
Additional context or constraints.
```

### PRD Format

```markdown
# [Feature Name]

## Overview
Brief description of the feature.

## User Stories
Links to related user stories.

## Requirements
Detailed requirements with acceptance criteria.

## Dependencies
Other features or systems this depends on.
```
