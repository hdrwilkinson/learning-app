## Phase 1: Input Gathering

### Topic & Goals (Conversational)

The AI chats with you to understand:
- **What** you want to learn ("Python for data science")
- **Why** you want to learn it ("Analyze sales data at work")

```
AI: What would you like to learn about?

You: Python for data science

AI: Great choice! What's your goal with Python?
    - Analyze datasets at work
    - Switch careers into data science
    - Build ML models
    - General data literacy

You: I want to analyze sales data at my job

AI: Perfectâ€”Python for workplace data analysis!
    Let me see what you already know...
```

The goal shapes the course content. "Python for data science" aimed at ML researchers would be different from the same topic aimed at business analysts.

### Baseline Assessment

A quick quiz determines your starting level (1-10 scale):

| Level | Description | Course Starts At |
|-------|-------------|------------------|
| 1-2 | Complete beginner | Very basics |
| 3-4 | Basic familiarity | Skip fundamentals |
| 5-6 | Intermediate | Working concepts |
| 7-8 | Advanced | Deep topics |
| 9-10 | Expert | Specialized content |

**Quiz mechanics:**
- 5-10 questions, progressive difficulty
- Three strikes (wrong answers) ends the quiz
- Your level = highest difficulty you answered correctly
- Can skip: "I'm a complete beginner" â†’ Level 1

**Why this matters:** A Level 5 learner doesn't need IPs on "What is a variable?" The AI generates content starting from *their* level.

### Why Sequential IP Generation?

IP content is generated *one at a time*, in order. Each IP has access to all previous IPs' content.

This enables:
- **Accurate prerequisites**: "This builds on what we covered in [previous IP]"
- **Consistent voice**: References feel natural
- **No orphan concepts**: Everything connects to what came before
